Infinite Scroll Documentation For use

What is Infinite Scroll ?

Infinite Scroll is a feature that loads the next set of posts automatically when visitors approach the bottom of the home page or posts page. It’s designed to help visitors read through many posts without effort.

Reference from: https://en.support.wordpress.com/infinite-scroll/

Why is Infinite Scroll needed ?

  • Navigation is enhanced
  • SEO is unharmed
  • Accessibility is maintained, as well as graceful degradation in finicky browsers
  • You can reduce page load size, but usually savings are compared to a page you wouldn’t make unless there was a solution like Infinite Scroll to tame it…

Reference from : http://www.sitepoint.com/ux-infinite-scroll-good-bad-maybe/

Default Options:

fetchItemsData: ''

noOfItems: 5

isServerDataFetch: false

careInitLoad: false

serverFetchParams: ""

objectProcessor: ""

loaderDOM: ""

scrollTopOnReload: false

delayStart: 0

Configuration Option Usage   fetchItemsData

fetchItemsData is the option to supply the input data to the InfiniteScroll Library.

The input can be

  • Direct Array
  • Local file system JSON based Data Fetch
  • Server side Data Fetch with server URL

Configuration Option Usage   noOfItems

noOfItems is the option to supply the number of items to be shown while mouse scrolling.

Configuration Option Usage   isServerDataFetch

isServerDataFetch is the option to inform the infiniteScroll library whether the data is available at Local File system or to fetch the data from server via ajax.

Configuration Option Usage   careInitLoad

careInitLoad is the option to inform the infiniteScroll library whether the initial data is to be loaded via this library or not.

Configuration Option Usage   serverFetchParams

serverFetchParams is the option to fetch the data periodically when scroll reaches end. This option should contain the query string to be attached with the domain URL.

Configuration Option Usage   objectProcessor

objectProcessor is one the key advantageous option in this infiniteScroll library. This option takes the result and process the result to fetch the array which is then supplied as the input. on any failure, appropriate errors will be thrown @ console.

Configuration Option Usage   loaderDOM

loaderDOM is the option which will be shown during data fetch activity and hidden on data fetch completion. The developer can advantage of decorating the loader DOM with CSS styles and their position.

Configuration Option Usage   scrollTopOnReload

scrollTopOnReload is the option to decide whether the page should be available with the same scrolled position or to be moved on to the top when the page is reloaded.

Configuration Option Usage   delayStart

delayStart is the option to create the delayed instantiation. The timing scale is measured in millisecond units.
Example: making delay for 3s = 3000ms.   <delayStart : 3000>

Global Methods  

$.infiniteScroll   init

  • This is the point of instantiation. Calling this method would re-instantiate the already configured DOM. Not Encouraged to use unless necessary.

$.infiniteScroll   avoidNSConflict

  • This Method avoids the namespace conflict by providing different name to work with the library APIs.

Option Configuration Methods  

resultData( )   returns function ( a, b, c ) { }

  • This method can be mapped to the user-defined methods to retrieve the result data received or the result data reached so far if the data fetch is periodically asynchronous.

domBuilder( )   returns function ( c ) { }

  • This method can be mapped to the user-defined methods to build or update the DOM with its contents.

Method Arguments Description

  • argument : a  The DOM which requested for data update
  • argument : b  Number of items currently supplied to update. A method to update a view.
  • argument : c  Overall Data Source available to be supplied as per configuration. A method to fetch the data source for any manipulation. Example: To count the number items rendered by fetching the data with several ajax calls on reaching the scroll end.

Useful Events  

scrollReachedLimit   message { a, b, c }

A jQuery Event is triggered when the scroll Reaches the limit or end but still possess data source to render.

scrollReachedEnd   message { a, b, c }

A jQuery Event is triggered when the scroll Reaches the End and no more data source is to be rendered.

Method Arguments Description

  • argument : a  Actual DOM Event
  • argument : b  Object: Event Type.
  • argument : c  Object: contains information (data items loaded so far, DOM Instance, Internal Element self scroll or Window scroll).

Internal Exceptional Handling  

Invalid Input   00X

  • Error Code : 001   If {isServerDataFetch:true}, {fetchItemsData} should contain only hostname (location.host), query string is to be separated it with {serverFetchParams}. So, Internal Error Code (Invalid Input Status Follows 00X) --> 001

  • Error Code : 002   Input Expected as (array or valid json). But supplied --> Invalid JSON type So, Internal Error Code (Invalid Input Status Follows 00X) --> 002

Unreachable Object Tree   0X0

  • Error Code : 010   Specified 'objectProcessor' in options --> {objectProcessor} is not found with the fetched data. So, Internal Error Code (Unreachable Object Tree Status Follows 0X0) --> 010

Network Data Status   X00

  • Error Code : 100   If Network Data Response From --> {fetchItemsData} ? {serverFetchParams} results with <success> status. So, Internal Error Code (Network Data Status -- Success -- Follows X00) --> 100

  • Error Code : 200   If Network Data Response From --> {fetchItemsData} ? {serverFetchParams} results with <failure> status. So, Internal Error Code (Network Data Status -- Failure -- Follows X00) --> 200
